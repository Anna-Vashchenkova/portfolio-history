= Договорёности по разработке

== Работа с Liquibase

* Базовый файл db.changelog-master.yaml создаётся в формате yaml и используется только для перечисления файлов, которые надо импортировать
* Для каждого changeset должен быть создан отдельный файл в папке src/main/resources/db/changelog/includes название файла должно соответствовать формату yyyy-MM-dd_PH-<НОМЕР-ЗАДАЧИ>_номер итерации.xml. Номер итерации вводится в формате двухзначного числа. т.е. если для задачи HP-12, 14 июня 2024 года создаётся первый элемент мирации, то имя файла должно быть `2024-06-14_HP-12_01`
